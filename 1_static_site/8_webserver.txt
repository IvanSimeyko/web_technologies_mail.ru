Вебсервер - сетевой сервер, который занимается обработкой протокола HTTP. Т.е. на одной стороне соединения клиент (часто браузер), на другой вебсервер.

Обзор: nginx, apache, loghttpd, IIS7
-----------
Демоны - программы которые запускаются вместе с ос, висят в памяти и обрабатывают данные которые приходят по сети.
------------
Краткое опиание работы вебсервера:
Вебсервера являются демонами и запускаются с помощью инит-скрипта sudo/etc/init.d/nginx start.
Вебсервера пишут логи, как правило доступны только супервпользователю.
При работе сервер понижает привелегии, чтобы если его взломали не смогли положить систему.
Запускает дочерние приложение. 
Готов к обработке запросов.

Файлы вубсервера:
- Конфиг /etc/nginx/nginx.conf
iclude /etc/nginx/sites-enables/*
- Init-скрипт /etc/init.d/nginx [start|stop|restart]
- PID-файл  /var/run/nginx.pid
-Error-лог /var/log/nginx/error.log
- Access-лог /var/log/nginx/access.log

Процессы веб сервера:
Master - работает от рута и это тот процесс который запускаем из init-скрипта. Выполняет:
- чтение и ваидация конфига
-открытие сокета и логов
- запуск и управляение дчерними процессами(worker)
- Graceful restart, Binary updates

Worker работает с привелегиями юзера и выполняет обрабобтку входящих запросов

Конфигурация веб сервера.
virual host, вирт.хост - секция конфига веб сервера, отвечающая за обслуживание определенного домена

location - секция конфига, отвечающая за обслуживание определенной группы урлов.

http - конфигурация для HTTP сервера
server - конфигурация домена (вирт.хоста)
server_name - имена доменов
location - лолкейшн, группа урлов
root, alias - откуда нужно брфть файлы
error_log - лог ошибок сервера
accesss_log - лог запросов
