Авторизация пользователей.
Нужно хранить информацию о пользователе в течение всего его сеанса работы. В протоколе HTTP нельзя свзать данные пользователя с его соединением (как это делается например при работе с базами данных). Поэтому браузер должен передавать серверу информацию о пользователе при каждом запросе.
Есть двв разнвх типа авторизации:
- basic авторизация:
- автризация основанная на куках.

BASIC HTTP AUTHORIZATION
Отмечаются локейшины требующие авторизации (напр. /admin), при запросе на этот урл сервер возвращает специальную ошибку 401(Unauthorized), браузер выводит специальное окно для авторизации, пользователь вводит туда данные и отправляет на сервер, который возвращает станичку или 403 (Forbidden - логин пароль не подходят). В дальнейшем браузер автоматически отсылает запросы на сервер с данными о пользователе при каждом запросе.

COOKIES
Cookies - небольшие фрагменты данных, которые браузер хранит на стороне клиента и передает на сервер при каждом запросе.
Cookies привязан к доменам, поэтому при каждом запросе сервер получает только свои cookies.
